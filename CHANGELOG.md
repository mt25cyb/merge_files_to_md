# 更新记录

本项目所有重要变更都会记录在此文件中，格式遵循 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.1.0/)，版本号遵循 [语义化版本](https://semver.org/lang/zh-CN/) 规范。

## [v2.2.0] - 2026-07-05
### 新增
- 新增 MD 文件自动分片拆分功能，配置项 `SplitMdMaxSize` 控制单分片最大KB
- 新增开关 `ExportWholeSingleFile`：分片模式下可额外输出完整不分片总文档
- 分片强制约束：单个源文件完整块不会跨分片截断，超大文件单独分片并添加警告提示
- 分片标准化排版：
  1. 所有分片首行为分片提示标注，后跟 `---` 分隔线
  2. 仅第一个分片携带 `[Title]` + `.header` 全局头部内容
  3. 非首分片无全局头部，直接展示文件内容
- 分片提示内文件列表自动换行，内置单行最多6个文件，无需额外配置
- 所有分片共享同一时间戳基准，命名格式 `前缀-时间_part1.md`
- 备份ZIP逻辑完全独立，不受MD分片功能任何影响

### 优化
- 重构Markdown片段预渲染逻辑，使用UTF-8编码字节精准计算分片尺寸，无预估偏差
- 统一所有文本块字节统计方式，兼容显示编码开关、空文件占位、多编码源文件转换场景
- 控制台新增分片分组统计输出，清晰展示总分片数量

### 兼容
- 原有全部功能完全向下兼容，旧版INI无需修改可正常运行
- `SplitMdMaxSize` 默认为0，不开启分片时保持原有单文件输出逻辑

## [v2.1.0] - 2026-07-04
### 新增
- 在Markdown二级文件标题后可选展示文件编码信息，INI新增开关 `ShowFileEncoding`
- 编码友好名称映射：`utf-8-sig` → `UTF-8 with BOM`、`gbk` → `GBK`、`latin-1` → `Latin-1 (兼容模式)`
- 标题展示格式示例：`## utils\demo.log (GBK)`

### 配置变更
- `[Options]` 节新增配置项 `ShowFileEncoding`，默认 false 关闭，完全兼容旧配置

### 优化
- 升级脚本版本常量为 `v2.1.0`
- 统一编码名称输出展示逻辑，消除原始编码名晦涩观感

## [v2.0.0] - 2026-07-04
### 重大变更
- 配置结构升级，原 `[Header]` 节拆分为 `[Title]` 节与外部 `.header` 文件，不向下兼容 v1.x 配置
- 主版本号升级，正式进入 v2 迭代周期

### 新增
- 新增 `[Title]` 配置节，用于设置合并文档的一级置顶标题
- 新增外部 `.header` 头部文件支持，原样输出 Markdown 内容并自动添加分隔线
- 新增正式版本号标识，控制台顶部显示工具版本与运行时间
- 头部文件不存在、内容为空时自动静默跳过

### 优化
- 统一所有 INI 配置节的注释规则，全部支持 `#` / `;` 整行注释
- 优化 Markdown 输出结构，层级更符合标准文档规范
- 调整控制台输出阶段划分，逻辑更清晰

## [v1.5.0] - 2026-07-04
### 新增
- 新增 `[IgnoreList]` 独立配置节，支持精确路径与 glob 通配符双重忽略规则
- 新增目录通配符 `目录\*` 语法，支持批量引入目录下直接文件
- 通配符匹配结果自动按文件名升序排列
- 通配符目录不存在、空目录、全被忽略时分别输出明确提示

### 优化
- 重构文件清单处理流程，统一走「展开 → 去重 → 过滤」流水线
- 忽略规则全局最高优先级，同时作用于手动条目与通配符展开
- 重复文件自动去重，保留首次出现的顺序

## [v1.2.0] - 2026-07-04
### 新增
- 新增同源文件备份 ZIP 压缩包功能，与 Markdown 同名同目录
- 新增 `[Options]` 配置节，`EnableBackupZip` 开关控制是否生成压缩包
- 压缩包完整保留原始相对目录结构
- 新增空文件自动占位处理，内容为空时显示 `(内容为空)`

### 优化
- 压缩包与 Markdown 使用同一份过滤后的文件列表，保持一致性
- 无有效文件时不生成空压缩包

## [v1.1.0] - 2026-07-04
### 新增
- 新增 `[LangMap]` 配置节，支持自定义文件名语言标识映射
- 新增代码块反引号自动转义，避免源文件内容导致 Markdown 格式错乱
- 新增文件名二级标题格式，文档结构更清晰
- 新增 INI 注释支持，配置节支持 `#` / `;` 整行注释
- Header 节内容原样保留，不误过滤注释字符

### 优化
- 扩展内置后缀语言映射表，新增 `.psm1`、`.env` 等特殊文件识别
- 优化 INI 解析逻辑，节顺序无强制要求

## [v1.0.0] - 2026-07-04
### 新增
- 初始版本发布，核心功能落地
- 支持 INI 配置读取，自定义输出目录与文件名前缀
- 支持多行文件头内容，代码块包裹输出
- 支持按清单顺序合并多个文件内容
- 支持 UTF-8 编码读取（兼容带/不带 BOM）
- 支持文件缺失自动跳过提示
- 支持输出目录自动创建
- 支持同名文件秒级时间戳防覆盖
